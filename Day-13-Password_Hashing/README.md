- Hashing means , the process by which we can make our passwords secure from attacks/leaks/decript 
 - In hashing we take users password and convert it into na non-readable string .    
 - We can make our own logics to make a function which can perform this task but we will not do that here , We will take use of a library named `bcrypt`


### The mechanism of hashing :-  
 - **Problem was** :- Multiple users can have same passwords. So their should be something which can differentiate between users .
 - So in hashing we don't put user's password directly in dB as a plane text like "hello123" ,
 - First We convert users password into a non-readable form for example "jdfnjknfnfndnfnjhnfj" by hashing algorithm & store it into a dB . 
 - But this will also  let to a problem : `same passwords = same converted form` , so to solve this this , concept to `salting`comes into picture .
 - Salting means attaching  a random string/data with the given password/data. 
 - Now user sends his credentials like username ,password ==> It gets to the backend and their before sending credentials to database  this password gets attached with the salt(generated by salting process ) and now this whole data i.e **`( password + salt ) is converted into a non-readable form by hashing algorithm and send to the database and it get stored their(salt & password both get stored into database)`** .
 - When we verify user for further endpoints then `we have to access both saved(hashed password and salt) this is the core concept this module`.
 - Unfortunately this salt automatically stored in hashed password so we don't have to access differently .
 - This library has  a mechanism/characterstics by which salt gets embedded into  hashed password  so when we use compare function  `bcrypt.compare(password,hashedpasword)` then we dont have to manually ask database for salt also.
 - Yeah we can separately store password & salt in database but why to do that , library has given u powers then why to store manually .

## Hash information     

Characters that are in the result of hash are :-   
`./ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789$.`

 Hash result  will be 60 characters long and it will contain the salt among other parameters for example:-

$[algorithm]$[cost]$[salt][hash]

- 2 chars hash algorithm identifier prefix. "$2a$" or "$2b$" indicates BCrypt
- Cost-factor (n). Represents the exponent used to determine how many iterations 2^n
- 16-byte (128-bit) salt, base64 encoded to 22 characters
- 24-byte (192-bit) hash, base64 encoded to 31 characters

Example:

> $2b$10$nOUIs5kJ7naTuTFkBy1veuK0kSxUFXfuaOKdOKf9xYT0KKIGSJwFa. 
 
 - 2b represents the hashing algorithm   
 - 10 reprensents the cost-factor:-2^10 rounds
 - salt = nOUIs5kJ7naTuTFkBy1veu
 - hash-value = K0kSxUFXfuaOKdOKf9xYT0KKIGSJwFa

 ## Rounds 
 > when you are hashing your data, the module will go through a series of rounds to give you a secure hash. *The value you submit is not just the number of rounds the module will go through to hash your data. The module will use the value you enter and go through 2 ^ rounds hashing iterations*.

 Means saltrounds= 3 , ==> 2^ 3 & our hashing algorithm(module) will go through these rounds
